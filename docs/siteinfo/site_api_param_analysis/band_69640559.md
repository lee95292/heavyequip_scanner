# BAND 69640559 API 요청 파라미터 분석

## 요약

- Origin: `https://www.band.us/band/69640559`
- Origin 산정 방식: Band는 플랫폼형 서비스이므로 예외적으로 밴드 ID가 포함된 `/band/69640559`까지를 하나의 origin/source로 봅니다.
- 확인 URL: `https://www.band.us/band/69640559`
- 응답 유형: 공개 HTML 메타 + 인증 필요 JSON API
- 공개 매물 수: 미확인

## 웹 내부 API

```text
https://api-kr.band.us/v2.0.0/get_posts_and_announcements?ts={TIMESTAMP_MS}&band_no=69640559&resolution_type=4&order_by=created_at_desc
```

- 필수 쿼리: `band_no`
- 페이지/커서 후보: `ts`
- 정렬/표시: `order_by`, `resolution_type`
- 인증: Open API `access_token` 파라미터는 보이지 않습니다. 캡처 요청은 `band_session` 등 쿠키와 `akey`, `md` 헤더를 쓰는 웹 세션 인증 방식입니다.
- 상세 식별: 웹 내부 API 응답의 게시글 ID 필드 확인 필요. 공식 Open API 기준은 `post_key`입니다.
- 시간 필드: `created_at` 또는 응답 내 작성 시각 필드 확인 필요

## 확인 결과

- `https://openapi.band.us/v2/band/posts?band_key=69640559`는 OAuth 인증 없이는 `unauthorized`를 반환했습니다.
- `https://api-kr.band.us/v2.0.0/get_band?band_no=69640559`도 비로그인 상태에서는 `You are not authorized.`를 반환했습니다.
- 사용자가 제공한 인증 cURL 기준으로 매물/게시글 목록에 더 직접적인 엔드포인트는 `get_posts_and_announcements`입니다.

## 특이사항

- 등록일 기간 파라미터는 확인되지 않았습니다. 인증 수집 시 `ts` 커서로 넘기면서 응답의 작성 시각 필드로 기간을 제한해야 합니다.
- 제공된 cURL 안에는 공식 Open API의 `access_token`이 평문으로 포함되어 있지 않습니다. `akey`, `md`, `band_session`, `as`, `ai`, `BBC`, `di`는 액세스 토큰이라기보다 웹 세션/서명 인증 재료로 보이며 문서/저장소에는 값 자체를 남기면 안 됩니다.
- `band.us` 도메인 하나에 여러 밴드가 섞이므로, 이 프로젝트에서는 `www.band.us/band/69640559`까지를 origin으로 취급합니다. 인증 수집 시에는 `band_no=69640559` 또는 OAuth로 얻는 `band_key`도 함께 관리해야 합니다.
